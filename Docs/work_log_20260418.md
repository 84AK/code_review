# 작업 로그 - 2026년 4월 18일 (2차 수정)

## 1. 개요
단일 HTML 파일 업로드 후 분석이 진행되지 않는 문제를 해결하기 위해 시스템 안정성 강화 및 필터링 로직 수정을 진행했습니다.

## 2. 긴급 수정 사항

### 2.1. 대소문자 구분 및 업로드 로직 보강 (`app.js`)
- **문제:** 파일 확장자가 대문자(`.HTML`, `.ZIP`)인 경우 필터링에서 제외되어 분석이 시작되지 않는 현상 발생.
- **해결:** `toLowerCase()`를 적용하여 모든 파일 확장자를 정상적으로 인식하도록 수정했습니다.
- **보강:** 분석 시작 시점에 현재 선택된 카테고리를 DOM에서 직접 다시 확인하도록 하여 인식 오류를 방지했습니다.

### 2.2. 블랙리스트 필터 완화 (`src/utils/zipHandler.js`)
- **문제:** 'docs' 키워드가 블랙리스트에 포함되어 있어, 학생이 파일명에 'docs'를 포함할 경우 분석 대상에서 제외되는 현상 발생.
- **해결:** 블랙리스트 키워드에서 `docs`를 제거하여 파일명 기반 필터 오작동을 해결했습니다.

### 2.3. AI 모델 고도화 (`src/utils/aiEvaluator.js`)
- **변경:** 모델 식별자를 `gemini-3-flash`에서 보다 최신 버전인 **`gemini-3.1-flash`**로 업데이트하여 분석 성능을 향상시켰습니다.

## 3. 결과 및 확인
- 단일 HTML 파일 업로드 시 즉시 "분석 중..." 상태로 전환되며 결과가 정상적으로 출력됨을 확인해야 합니다.
- 만약 여전히 문제가 발생한다면, 브라우저의 '새로고침'을 한 번 진행한 후 다시 시도해 보시길 권장합니다.
