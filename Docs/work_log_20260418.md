# 작업 로그 - 2026년 4월 18일

## 1. 개요
POSE 게임 분석 시스템에 단일 HTML 파일 업로드 기능을 추가하고, 샘플 코드와의 유사성(복제 여부)을 평가 항목에 반영했습니다. 또한 AI 모델을 최신 버전으로 업데이트했습니다.

## 2. 주요 수정 사항

### 2.1. 단일 HTML 업로드 지원 (`app.js`, `index.html`)
- **수정 전:** `.zip` 파일만 필터링하여 처리하며, 반드시 압축 해제 과정을 거쳐야 했습니다.
- **수정 후:** 
  - `handleFiles` 함수가 `.zip`과 `.html` 파일을 모두 허용하도록 확장되었습니다.
  - 단일 HTML 파일의 경우 압축 해제 없이 즉시 텍스트를 읽어 분석(`parseStudentFiles`)으로 전달합니다.
  - UI 안내 문구 및 파일 선택 `accept` 필터에 `.html`을 추가했습니다.

### 2.2. 샘플 복제 여부 평가 로직 추가 (`src/data/referenceData.js`, `src/utils/aiEvaluator.js`)
- **기준 데이터 추가:** `extracted_pose/game_sample` 폴더의 3가지 게임(Airplane, Car, Snake)에 대한 핵심 구조와 특징을 `referenceData.js`에 명시했습니다.
- **프롬프트 강화:** AI가 학생의 코드를 분석할 때, 위 3가지 베이스라인과 코드가 거의 똑같은지(단순 복제 여부)를 필수적으로 체크하게 했습니다.
- **감점 규칙:** 단순 복제인 경우 등급을 C~D로 하향 조정하고, 교육적 조언을 피드백에 포함하도록 지시했습니다.

### 2.3. AI 모델 업데이트 (`src/utils/aiEvaluator.js`)
- 사용성 지침에 따라 모델을 `gemini-2.5-flash`에서 **`gemini-3-flash`**로 업데이트했습니다.

## 3. 결과 및 확인
- 이제 사용자는 POSE 게임 카테고리에서 단일 `.html` 파일을 업로드하여 분석할 수 있습니다.
- 분석 결과, 샘플 코드와 똑같이 제출한 학생은 독창성 점수에서 감점을 받게 됩니다.
